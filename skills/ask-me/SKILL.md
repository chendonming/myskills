# ask-me 技能

## 概述

用户描述任务时，经常存在隐含的假设、未说明的偏好、或多种可能的方案。
这些歧义在执行前用户自己也没意识到，等到 AI 做完了才发现不对——浪费大量时间。

这个技能的作用是：**在动手之前，通过多轮渐进式澄清，确保AI和用户的思维完全对齐。**

核心方式是 **渐进式澄清循环**：
- 每轮提出 **1-3个** 当前最有价值的问题
- 用户回答后，评估是否还有歧义
- 如果有，继续下一轮提问（仍然1-3个）
- 直到确认没有任何歧义，再开始执行

## 工作流程

### Step 1: 理解任务

收到 `/ask-me` 调用时，分析用户附带的描述内容以及当前对话上下文（最近几轮对话），
彻底理解用户想完成什么目标。不要急于提问——先确保你理解了任务的整体轮廓。

特别注意识别：
- **模糊的目标**：用户说"优化一下这个页面"——优化性能？UI？代码可维护性？
- **未指定的细节**：用户说"加个导出功能"——导出什么格式？全部数据还是筛选后的？
- **隐含的假设**：用户说"修一下这个bug"——他们可能假设你已知bug现象，但他们没说清楚
- **方案选择点**：任务可以用多种方式实现，哪种最适合用户场景？

### Step 2: 提出1-3个问题（每轮）

基于当前的理解程度，使用 `AskUserQuestion` 工具提出最有价值的 **不超过3个** 问题。
一次性展示过多问题会让人感到困惑和压力。`AskUserQuestion` 的 `questions` 数组支持同时提交1-4个问题，每轮最多使用3个。

**提问原则：**

- **具体而非抽象**：不要问"你想要什么？"这种太宽泛的
  - ❌ "您的需求是什么？"
  - ✅ "这个导出功能你希望输出 CSV 还是 Excel 格式？"

- **提供选项而非开放题**：尽量给出明确选项让用户选择。利用 `AskUserQuestion` 的 `options` 字段提供互斥选项，精简描述即可让用户快速决策
  - ❌ "你想怎么处理错误？"
  - ✅ `AskUserQuestion` 调用时，`options` 设置为 `[{label: "跳过错误行继续", description: "记录错误日志不影响后续导入"}, {label: "遇到错误立即停止", description: "确保数据完整性"}]`

- **解释"为什么问"**：在 `question` 字段中简短说明这个问题为什么重要
  - ✅ `question` 设置为："导出格式你希望用 CSV 还是 Excel？CSV 通用性好，Excel 支持多Sheet和格式。"

- **聚焦关键决策**：只问那些真正影响方案设计和实现方向的问题
  - 不要问可以通过查代码库自行确认的信息
  - 不要问和任务无关的细节

- **优先级排序**：把当前最重要的1-3个问题放在最前面。如果还有什么想确认的但优先级不高，留到下一轮再问。

- **必须使用 AskUserQuestion 工具**：所有向用户提出的问题都必须通过 `AskUserQuestion` 工具提交，不得直接在文本中提问。利用其结构化能力（`question`、`options`、`multiSelect` 等字段）让用户高效决策。

### Step 3: 等待回答并评估

用户回答当前轮次的问题后（`AskUserQuestion` 工具会返回用户的答案）：

1. **消化回答**：理解用户的选择和偏好，从 `answers` 中提取用户的选项或自定义输入
2. **评估状态**：目前的认知还有没有明显的歧义或未确认的决策点？
3. **决定下一步**：
   - **仍有歧义** → 回到 Step 2，使用 `AskUserQuestion` 提出下一轮的1-3个问题
   - **已无歧义** → 进入 Step 4

注意：即使你已经有了5个问题想问，也必须拆成多轮，每轮最多3个。
先问最重要的，下一轮再问次重要的。

### Step 4: 总结对齐并执行

所有歧义消除后：

1. **总结对齐结果**：简明扼要地总结已确认的所有关键决策点
2. **声明开始执行**：直接开始执行原始任务，不需要再问"那我开始喽？"

例如：

> 好的，已确认所有关键点：
> 1. 导出格式：CSV
> 2. 数据范围：当前筛选结果
> 3. 错误处理：跳过错误行，记录日志到 `errors.log`
>
> 没有其他歧义了，开始实现。

## 完整流程示例

```
用户: /ask-me 我想给后台加个导入用户的功能

AI: 调用 AskUserQuestion 工具，提交两个问题：
    [
      {
        question: "导入的文件格式是 CSV 还是 Excel？这决定我用哪个解析库。",
        header: "文件格式",
        options: [{label: "CSV"}, {label: "Excel"}],
        multiSelect: false
      },
      {
        question: "发现有重复用户（如相同邮箱），如何处理？",
        header: "重复处理",
        options: [
          {label: "跳过重复", description: "记录日志继续导入"},
          {label: "覆盖更新", description: "新数据替换旧数据"},
          {label: "报错停止", description: "遇到重复立即中断"}
        ],
        multiSelect: false
      }
    ]

用户: (选择 CSV 和 跳过重复)

AI: 调用 AskUserQuestion 工具，再问一个问题：
    [
      {
        question: "导入的用户需要分配到某个部门吗？还是在全局用户池？",
        header: "部门归属",
        options: [
          {label: "指定部门", description: "CSV 中包含 department_id 列"},
          {label: "全局用户池", description: "不分配部门"}
        ],
        multiSelect: false
      }
    ]

用户: (选择"指定部门")

AI: 明白了。确认我的理解：
    • 格式：CSV
    • 重复处理：跳过并记录日志
    • 部门字段：CSV 中包含 department_id 列

    没有歧义了，开始实现导入功能。
```

## 关键提醒

- **只做对齐，不做其他**：此技能的唯一任务是消除歧义。在进入 Step 4 之前，不要修改任何代码或文件。
- **问题要有区分度**：问那些真正能改变方案设计的问题。如果无论用户选A还是选B你的做法都一样，那这个问题就不值得问。
- **保持对话自然**：不要用模板化的语言。根据任务上下文自然地提问。
- **不要过度分析**：如果任务已经非常清晰（如"把第42行的 'foo' 改成 'bar'"），就没有必要调用这个技能。
  调用此技能本身就是一次主动判断——你觉得可能有歧义才用，不是所有任务都需要。
- **控制每轮数量**：任何时候向用户展示的问题都不超过3个。宁可分多轮，不要一次性全抛出来。
